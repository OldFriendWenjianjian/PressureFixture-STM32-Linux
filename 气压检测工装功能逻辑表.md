## 2. 基本逻辑

- 设备包含 6 个气罐，分为 3 个标定气罐和 3 个测试气罐。
- 标定气罐压力点为 `50mmHg / 150mmHg / 250mmHg`。
- 测试气罐压力点为 `100mmHg / 200mmHg / 285mmHg`。
- PCBA 气密性测试、标定、结果测试均为 `8 路 PCBA 同步测量`。
- 主控通过 `UART1 -- UART8` 对应 `通道1 -- 通道8`，串口通信采用快速分时复用。
- 虽然串口按 8 路轮询发送，但 8 路 PCBA 处于同一压力平台、同一阶段、同一真实压力值下完成测量。
- 进入 PCBA 气密性测试、标定、结果测试后，所有气罐进气口关闭，只保留当前压力点对应气罐向 8 路 PCBA 共同供气。
- 当前压力稳定后，主控向 8 路 PCBA 下发当前真实压力值，再完成该压力点的测量、标定或测试。

## 4.1 串口协议规则
- MCU 为主机，8 路 PCBA 为从机。任何时候只能由主机主动发起，从机只能在收到指令后立即返回。
- `UART1` 至 `UART8` 对应 PCBA 通道 1 至通道 8。
- 波特率固定为 `115200`。
- 数据格式固定为 `8N1`，即 8 数据位、无校验、1 停止位。
- MCU 采用轮询方式依次向通道 1 至通道 8 发送命令。
- MCU（主板）下发指令时，从机应立即回包，最大回包等待时间10ms。

包结构如下：

| 字段 | 字节数 | 说明 |
| --- | ---: | --- |
| `55 AA` | 2 | 固定帧头 |
| `CMD` | 1 | 命令字 |
| `CH` | 1 | 通道号，`01` 至 `08` |
| `LEN_L LEN_H` | 2 | `DATA` 长度，小端 |
| `DATA` | N | 参数或返回数据 uint32 little-endian |
| `CRC_L CRC_H` | 2 | `CRC16/MODBUS`，小端 |

CRC 校验范围为 `CMD + CH + LEN + DATA`，不包含帧头 `55 AA`。CRC 初值 `0xFFFF`，多项式 `0xA001`。

气压值使用 4 字节小端整数，单位为 `0.001mmHg`。命令字：

| 命令 | 名称 | 方向 | DATA | 说明 |
| --- | --- | --- | --- | --- |
| `01` | 进入测试模式 | MCU -> PCBA | 无 | 开机后下发 |
| `02` | 记录零点 | MCU -> PCBA | 无 | 气压未上压前下发 |
| `03` | 开机 | MCU -> PCBA | 无 | 开机 |
| `04` | 查询低电状态 | MCU -> PCBA | 无 | 切换 4.5V 后查询 |
| `05` | 查询正常状态 | MCU -> PCBA | 无 | 恢复 5V 后查询 |
| `10` | 标定压力同步 | MCU -> PCBA | 气压值 4 字节 | 50/150/250mmHg 标定阶段 |
| `11` | 测试压力查询 | MCU -> PCBA | 无 | 100/200/285mmHg 测试阶段，要求 PCBA 回传 4 字节测值 |
| `20` | / | MCU -> PCBA | 无 | 结果显示阶段可用 |
| `7F` | 应答包 | PCBA -> MCU | 无或 1 字节 | PCBA 对主控指令的应答包 |

## 5. 8 路同步执行规则

1. PCBA 气密性测试、标定、结果测试均使用同一个压力平台，8 路 PCBA 同步测量。
2. 每个测量阶段开始前，关闭全部气罐进气阀，不允许边测量边补气。
3. 当前阶段只允许对应压力点气罐的出气阀打开，向阀13--20后的 8 路 PCBA 同时供气。
4. 必须等压力检测1--6中的对应气罐压力稳定，且压力检测7--14全部稳定后，主控才发送串口测量指令。
5. 主控通过 UART1--8 快速轮询发送同一阶段指令和同一真实压力值，8 路 PCBA 在同一压力条件下完成测量。
6. 任一路 PCBA 未响应、响应异常或测量失败，该压力点判定为失败，不进入下一个压力点。
